Documentation & Guide

Getting Started

Langkah pertama menggunakan RepoFlow untuk mengelola repository GitHub Anda.

  • Buat Personal Access Token di GitHub
  • Login dengan username dan token
  • Mulai kelola repository

Create Repository

Membuat repository baru di GitHub dengan mudah.

  • Klik menu "Create Repo"
  • Isi nama repository
  • Tambahkan deskripsi (opsional)
  • Pilih Public/Private

Delete Repository

Menghapus repository dengan konfirmasi keamanan.

  • Klik menu "Delete Repo"
  • Masukkan nama repository
  • Konfirmasi nama
  • Klik tombol delete
⚠️ Tindakan ini permanen!

Upload Mode 1: Multiple Files

Upload banyak file sekaligus ke repository.

  • Pilih tab "Mode 1: Multiple Files"
  • Pilih target repository
  • Pilih file (bisa multiple)
  • Klik "Upload Files"

Upload Mode 2: ZIP Archive

Upload file ZIP, auto extract, dan upload semua file.

  • Pilih tab "Mode 2: Archive (ZIP)"
  • Upload file ZIP
  • Tunggu auto extract
  • Klik "Extract & Upload"

Live Terminal

Terminal real-time untuk melihat proses operasi.

  • Muncul otomatis saat operasi
  • Warna log: Sukses, Error, Warning
  • Progress bar animasi
  • Bisa ditutup manual

Frequently Asked Questions

Bagaimana cara mendapatkan Personal Access Token (PAT)?
Buka GitHub Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token. Centang permission: repo, delete_repo. Copy token yang dihasilkan.
Apakah token saya aman?
Ya, token hanya disimpan di memori browser dan tidak pernah dikirim ke server manapun selain API GitHub.
Berapa batas upload file?
GitHub membatasi file individual maksimal 100MB. Untuk file besar, disarankan menggunakan Git LFS.
Apakah bisa upload folder dengan struktur?
Untuk struktur folder, gunakan Mode 2 (ZIP Archive). ZIP file akan diekstrak dan mempertahankan struktur folder.
